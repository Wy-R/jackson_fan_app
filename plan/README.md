# 数据存储改造计划

把 mock 阶段写死的数据,改造为「本地 + 远端」两套真实存储。

## 总原则:存储位置跟着「数据性质」走

| 数据 | 性质 | 存储位置 | 方案 |
|---|---|---|---|
| 收藏 | 用户私有 | 本地 | shared_preferences + JSON 快照 |
| 树洞 | 用户私有 | 本地 | shared_preferences + JSON(量大再迁 sqflite) |
| 每日一言 | 全员共享、只读 | 远端 | jsDelivr CDN 拉 JSON + assets 兜底 |
| 作品 / 图集 / 艺人资料 | 全员共享、只读 | 远端 | 同上(后续) |

## 实施顺序(风险从低到高)

1. [01-收藏-本地](01-favorites-local.md) — 纯本地、改动小、能立刻看到效果
2. [02-树洞-本地](02-treehole-local.md) — 纯本地、新功能、不影响现有代码
3. [03-每日一言-远端](03-daily-remote.md) — 涉及网络,需兜底,最后做
